Cerebrus User Guide

Cerebrus is a comprehensive GUI toolkit for Unreal Engine Android profiling workflows. It simplifies the process of collecting logs, moving files, and generating performance reports.

Getting Started

Launch Cerebrus to see the main dashboard. The application will automatically check for and install ADB (Android Debug Bridge) if it's missing from your system. The interface is divided into several sections: - Menu Bar: Access file operations, views, tools, settings, and help. - Profile Summary: Displays the currently loaded profile information. - Device List: Shows connected Android devices. - Data and Perf Report: Configure paths and perform bulk actions. - Logging: View real-time application logs.

Usage / Steps to Use

This section provides step-by-step instructions for common workflows in Cerebrus.

Workflow 1: First-Time Setup

  1. Launch Cerebrus - The application will automatically check for ADB. If not found, it will be installed automatically.
  2. Create or Load a Profile - Go to File → New Profile or use the default "Titan" profile.
  3. Configure Package Name - In the Edit Profile dialog, enter your Unreal Engine app's package name (e.g., com.lightfury.titan).
  4. Set Output Path - Choose a workspace folder where all data and reports will be saved (defaults to C:\).
  5. Save Profile - Your settings will be auto-saved to the profile JSON file.

Workflow 2: Collecting Data from a Device

  1. Connect Your Android Device - Enable USB debugging and connect via USB cable.
  2. List Devices - Click the "List Devices" button to scan for connected devices.
    Note: If this fails on first run, open Command Prompt and run adb devices manually once to initialize the ADB daemon.
  3. Select Device - Click on the device row in the table to select it. The device make/model will be automatically appended to your output path.
  4. Verify Package Installation - Check that the "Package Found" column shows "Yes" for your device.
  5. Run Your App - Launch your Unreal Engine application on the device.
  6. Start Profiling (Optional) - Click "Start Profiling" to begin collecting performance data.
  7. Perform Testing - Use your app and perform the actions you want to profile.
  8. Stop Profiling (Optional) - Click "Stop Profiling" when done.
  9. Move Data to PC:
    • Check "Move Logs" to copy log files from the device
    • Check "Move Profiling Data" to copy CSV profiling data
    • Files will be organized in Output Path/DeviceMakeModel/Logs and Output Path/DeviceMakeModel/CSV

Workflow 3: Generating Performance Reports

  1. Ensure Data is Available - Make sure you have CSV files and/or log files in your output path (either from moving data or from previous sessions).
  2. Set Output File Name - Enter a descriptive name for your report in the "Output File Name" field.
  3. Choose Report Type:
    • Generate Perf Report Only - Processes CSV files using PerfreportTool.exe to create performance reports
    • Generate Colored Logs Only - Converts text logs to color-coded HTML files for easier reading
    • Generate Perf Report + Colored Logs - Performs both operations together
  4. Click the Generate Button - The selected operation will run and output files will be created in the device-specific subfolder.
  5. View Results - Click "View HTML Logs" to open the output folder and view generated reports.

Workflow 4: Working with Multiple Devices or Projects

  1. Create Separate Profiles - Go to File → New Profile for each project/app.
  2. Configure Each Profile - Set unique package names and output paths for each.
  3. Save Profiles - Use descriptive filenames (e.g., titan_profile.json, game2_profile.json).
  4. Switch Between Profiles - Use File → Open Profile to load different configurations.
  5. Device Auto-Organization - When you select different devices, Cerebrus automatically creates device-specific subfolders to keep data organized.

Workflow 5: Analyzing Existing Data

  1. Set Output Path - Point to a folder that already contains Logs and/or CSV subdirectories.
  2. Generate Reports - You can generate performance reports and colored logs even without moving data from a device.
  3. No Device Required - This workflow works entirely offline with existing data.

Profile Management

Profiles allow you to save and load configurations for different projects or devices. - New Profile: Create a fresh configuration. - Open Profile: Load an existing JSON profile. - Edit Profile: Modify the current profile settings. - Auto-Save: Changes to paths and settings are automatically saved to the current profile.

Device Management

The device list shows all connected Android devices. - Refresh: Click "List Devices" to scan for connected devices. - Selection: Click on a device row to select it for operations. - Status: The table shows the device model, serial number, Android version, and if the target package is installed.

Remote Profiling

Control Unreal Engine profiling directly from Cerebrus. - Start Profiling: Starts profiling data on selected device if Package is running actively in foreground. - Stop Profiling: Stops profiling data on selected device if Package is running actively in foreground. - Note: The application must be running on the selected device for these commands to work.

File Actions

Manage files between your PC and the connected Android device.

Configuration

  • Output File Name: The base name for generated files.
  • Output Path: The main workspace folder. Files moved from devices will be saved here, and generated reports will be output here.
  • Automatic Device Organization: When a device is selected, a subfolder with the device make and model is automatically created in the Output Path to keep your data organized.

Bulk Actions (Phone to PC)

  • Move Logs: Copies logs from Saved/Logs on the device to your PC.
  • Move Profiling Data: Copies profiling data from Saved/Profiling/CSV on the device to your PC.

Report Generation

Process collected data into readable formats.

Bulk Actions (PC to PC)

  • Generate Perf Report Only: Runs PerfreportTool.exe on CSV files.
  • Generate Colored Logs Only: Converts text logs to color-coded HTML files.
  • Generate Perf Report + Colored Logs: Performs both operations in sequence.
  • View HTML Logs: Opens the output folder to view generated HTML logs.

Settings

Customize your experience via the Settings menu. - Load Theme: Switch between Dark, Light, or System themes. - Log Colors: Customize the colors used in the application log window. - Key Bindings: View or edit keyboard shortcuts.

Troubleshooting

  • No Devices Found: Ensure USB debugging is enabled and ADB is running.
  • List Devices Not Working: If the "List Devices" button does not work automatically, open a command prompt and run the command adb devices manually once to initialize the daemon.
  • Tool Not Found: Verify that PerfreportTool.exe path is correctly configured in your environment or settings.
